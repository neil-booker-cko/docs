# IP Network Ranges Reference

Special-purpose and reserved IP address ranges defined by IANA and various RFCs.
These ranges are not routable on the public internet (unless otherwise noted) and
are used for private networking, documentation, testing, and protocol functions.

## RFC 1918 Private Ranges

The three private address blocks from RFC 1918. Any organisation may use these
without IANA coordination. They must not be advertised to the public internet.

| Block | Range | Addresses | Typical Use |
| --- | --- | --- | --- |
| `10.0.0.0/8` | `10.0.0.0` – `10.255.255.255` | 16,777,216 | Enterprise, data centre |
| `172.16.0.0/12` | `172.16.0.0` – `172.31.255.255` | 1,048,576 | Enterprise, VMs |
| `192.168.0.0/16` | `192.168.0.0` – `192.168.255.255` | 65,536 | SOHO, home networks |

---

## CGNAT — Shared Address Space (RFC 6598)

`100.64.0.0/10` (`100.64.0.0` – `100.127.255.255`, 4,194,304 addresses) is reserved
for use between ISP customer premises equipment (CPE) and carrier-grade NAT (CGNAT)
devices. It must not be used on the public internet or within enterprise networks.
Unlike RFC 1918, this space is specifically reserved for ISP infrastructure only.

### Unique Local Addresses (ULA — RFC 4193)

ULA is the IPv6 equivalent of RFC 1918. The `/7` block splits into:

| Prefix | Status |
| --- | --- |
| `fc00::/8` | Reserved (centrally assigned, not yet used) |
| `fd00::/8` | Locally assigned — generate a random `/48` from this range |

A ULA `/48` is generated by setting the L bit (`fd`) and filling bits 16–63 with a
40-bit pseudo-random value. This minimises collision risk when merging networks.

## Notes

- **Bogon filtering** — BGP edge routers should filter advertisements of RFC 1918,
  RFC 6598, documentation, and other special-purpose ranges both inbound and
  outbound. Team Cymru and other organisations publish bogon prefix lists.

- **Martian packets** — packets arriving on an interface with a source address that
  should not be reachable from that direction (e.g. RFC 1918 arriving from the
  internet). Discarded by `ip verify unicast source reachable-via rx` (uRPF strict)
  on Cisco or equivalent.

- **`198.18.0.0/15`** (RFC 2544 benchmarking) is sometimes mistakenly used for lab
  addressing. Avoid — it overlaps with ranges that some ISPs may actually route.
